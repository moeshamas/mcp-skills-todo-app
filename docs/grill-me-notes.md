# grill-me Notes

## Prompt Used
You are using the "grill-me" skill.

I am building a small React to-do app for an assignment.

The app allows users to:
- add tasks
- mark tasks as completed
- delete tasks
- filter tasks by All / Active / Completed

Grill this plan.

Tell me:
- what is unnecessary
- what is missing
- what risks I should watch for
- how to keep it simple and appropriate for the assignment

## Key Feedback
- Keep the project limited to the required core features.
- Do not add login, backend, database, drag-and-drop, categories, or extra UI features.
- Add one small missing rule: prevent empty or whitespace-only task submission.
- Define clear acceptance criteria for add, complete, delete, and filter behavior.
- Keep labels and UI simple so testing and Playwright demo are easy.
- Do not refactor too early; first keep the app working, then improve architecture later as required.
- Make sure GitHub Issues clearly map to features and testing steps.

## Scope Decisions After Feedback
- Keep only the core features.
- Prevent empty task submission.
- Do not add backend, database, login, or extra features.
- Keep the app simple and focused on the assignment workflow.

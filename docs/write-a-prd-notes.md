# write-a-prd Notes

## Prompt Used
You are using the "write-a-prd" skill.

Write a simple PRD for a small React to-do app for an assignment.

The app allows users to:
- add tasks
- prevent empty or whitespace-only task submissions
- mark tasks as completed
- delete tasks
- filter tasks by All / Active / Completed

Keep the scope intentionally small.

Include:
- product overview
- goals
- non-goals
- target user
- functional requirements
- acceptance criteria
- technical notes
- out-of-scope items

## Summary of Output
- A PRD was created defining a simple React to-do app
- The scope was intentionally kept small
- Clear functional requirements and acceptance criteria were defined

## Key Decisions
- Focus only on core to-do features
- Prevent empty task submissions
- No backend, login, or database
- PRD will be used to generate GitHub Issues

# Skills Used

## grill-me
Critically evaluate a plan. Identify unnecessary parts, missing elements, risks, and simplifications.

## write-a-prd
Generate a Product Requirements Document including overview, goals, non-goals, users, requirements, and acceptance criteria.

## prd-to-issues
Convert a PRD into structured GitHub Issues with clear titles, descriptions, and acceptance criteria.

## tdd
Guide development using test-driven development: define tests, then implement features to pass them.

## improve-codebase-architecture
Refactor code to improve structure, readability, and maintainability without overengineering.
